Pista

User Guide

Overview

Pista is a simple way to keep track and manage your tasks. It is created with one goal in mind - simplicity. All of our functions can be performed using command line interface and your tasks are available to view in Calendar that paired with To-do list view. It provides a way to schedule and manage your time fast and efficient.

Pista is built using Java and will be a standalone program for Windows.

List of Commands

The following are commands that the user can use to perform necessary functions in our program.

  1. Add
  2. Edit
  3. Delete
  4. List
  5. Search
Add

Synopsis

add <title> [startDate] [startTime] [endDate] [endTime] [repeat] [delayType recur]

<field> - field in arrow brackets are a compulsory field
[parameters] - fields in square brackets are optional parameters

Description

This command will add a new task.

Complusory

<title>

e.g. "Hello World"
Make sure the title is enclosed by double quotations.

Parameter

[startDate] / [endDate]

  • Default value: system date
  • To enter a day of the week, enter the first three letters of the day (i.e. "mon", "thu", or "sun"). By default, the system will recognise the entry is meant for current week.
  • To specify a date, enter the date in the given format: DDMMYY

    • [startTime] / [endTime]

  • Default start time: current system time
  • Default end time: 2359 for the current system date
  • General format: HHMM

    • [repeat]

  • Short hand format to specify if the task reoccurs.
    -n : non reoccurring task
    -r : reoccurring task

    • [delayType recur]

  • delaytype: To specify type of delay.
    -d : day
    -w: week
    -m: month
    -y : year
  • recur: numeric value to specify number of recurrence
    e.g : -1 : 1 day/month/year
    -8 : 8 days/months/years

    • Edit

    Synopsis

    edit <taskId> to [newTitle] [newDate] [newTime[duration]] [newRepeat] [delayType recur]

    <field> - field in arrow brackets is a compulsory field
    [parameters] - fields in square brackets are optional parameters

    Description

    This command will edit current saved task. Return error if the system is unable to find the task.

    Complusory

    <taskId>

    Task id will be the number that is assigned to the task.

    Parameter

    [newTitle]

  • Default value: system date
  • If a new title is not specified in the command, system will assume the title remains the same
  • Specify a new title will replace the old title of the task

    • [newDate]

  • Default start time: current system time
  • Default end time: 2359 for the current system date
  • If this is not specified in command, system will assume the date remains the same
  • To enter a day of the week, enter the first three letters of the day (i.e. "mon", "thu", or "sun"). By default, the system will recognise the entry is meant for current week.
  • To specify a date, enter the date in the given format: DDMMYY

    • [newTime]

  • If time is not specified in command, system will assumes the time remains the same
  • Default start time: current system time
  • Default end time: 2359 for the current system date
  • General format: HHMM

    • [newRepeat]

  • If this is not specified in command, the system will assume that the value remains the same.
  • Short hand format to specify if the task reoccurs.
    e.g: -n : non reoccurring task
    -r : reoccurring task

    • [delayType recur]

  • delaytype: To specify type of delay.
    -d : day
    -w: week
    -m: month
    -y : year
  • recur: numeric value to specify number of recurrence
    e.g : -1 : 1 day/month/year
    -8 : 8 days/months/years

    • Command Example

    edit 123 to Department Meeting with Sales fri every 2 week

    In the above sample, this task with an ID - 123 will be changed to a new title called "Department Meeting with Sales", on this Friday, with the time unchanged for every 2 weeks.

    Delete

    Synopsis

    delete <taskId>

    [parameters] - fields in square brackets are optional parameters
    <title> - field in arrow brackets is a compulsory field

    Description

    This command will delete selected saved task.
    Return error if the system is unable to find the task.
    If the task has repetitions, all repetitions will be deleted.

    Complusory

    <taskId>

    Task id will be the number that is beside the task title in the list view

    List

    Synopsis

    list [date] [time]

    [parameters] - fields in square brackets are optional parameters

    Description

    This command will list all tasks within the specific date and time.
    Return empty if the system is unable to find any tasks.

    Search

    Synopsis

    find [keyword]

    [parameters] - fields in square brackets are parameters.

    Description

    This command will search for tasks that contains the keyword.
    Return empty if the system is unable to find any tasks.

    Parameter

    [keyword]

  • If user did not specify the keyword, system will assume the keyword is universal (every alphanumeric)
  • Specify short phrases
